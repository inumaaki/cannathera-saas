# Cannathera — Internationalization (i18n)

**Client priority #1.** Translations are DATA, not code.

## Locales
Single source: `packages/shared/src/i18n.ts`.
- Supported: `de, en, tr, bg, ar`
- Default / pilot: **`de`** (German)
- RTL: **`ar`** (Arabic)

## Stack
- **next-intl v4** on Next.js 16 App Router.
- Locale-prefixed routing (`localePrefix: "always"`): `/de`, `/en`, `/tr`, `/bg`, `/ar`.
- Files (`apps/web/src/i18n/`):
  - `routing.ts` — `defineRouting` from shared LOCALES/DEFAULT_LOCALE
  - `navigation.ts` — locale-aware `Link`, `redirect`, `useRouter`, `usePathname`
  - `request.ts` — `getRequestConfig`; **deep-merges locale messages over German base** so missing keys fall back to `de` (never breaks UI). Skips `_`-prefixed meta keys.
  - `middleware.ts` — `createMiddleware(routing)`, matcher excludes api/_next/static.
- Plugin: `next.config.ts` wraps with `createNextIntlPlugin("./src/i18n/request.ts")`.

## Static UI strings
- Catalogs: `apps/web/messages/{locale}.json` (ICU format).
- **Zero hardcoded UI strings** — every label/button/validation from catalogs.
- `de.json` = canonical/complete. `en.json`, `ar.json` = real translations. `tr.json`, `bg.json` = placeholders (fall back to `de` until M10).

## Dynamic content (questionnaires, reports)
Translatable **in the DB**, not just static UI — this is what lets a patient fill a German-authored form in Turkish/Arabic:
- `QuestionTranslation(questionId, locale, label, helpText)`
- `OptionTranslation(optionId, locale, label)`
- Canonical German lives on `Question.label` / `QuestionOption.label`; missing locale → falls back to canonical.

## RTL
- `ar` → `<html dir="rtl">` (set in `[locale]` root layout).
- Layout uses **logical CSS properties** (`ps-`/`pe-`, `start`/`end`, `ms-`/`me-`) so RTL flips cleanly. No hardcoded left/right.

## Formatting
Locale-aware dates/numbers (medical dates, dosages) via `Intl` / next-intl formatters.

## Pilot rule
Patient-facing views ship **100% German**. Other locales render with fallback + correct RTL. Full translation = Module M10.
